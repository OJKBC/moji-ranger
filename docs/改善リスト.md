## もじもじアトラクション＋さんすうバトル＋ライフ制（2026-07-08）

- **⑤ もじもじアトラクション**（旧「ねこ」をつくれ！・ID `word-neko` 不変）: 単語プールを `src/data/words.ts` に分離（1行追加で単語を増やせる）。難易度=文字数（Lv1=2文字/Lv2=3文字/Lv3=4文字）。ラウンドごとにシャッフル列から出題。ディストラクターは pickDistractors に `exclude`（単語の構成文字）を追加して動的生成（Lv2+は苦手ペア優先）。完成時は単語読み上げ＋星バースト追加
- **⑥ 「3」をさがせ！を非表示**: `Stage.hidden` フラグで StageMap/「つぎ」判定から除外。コード・データ・進捗は温存。さんすうの解放条件は「もじもじ Lv1」に付け替え
- **⑦ さんすうバトル**（旧「2+1」ゲート！・ID `math-add-1` 不変）: `mathLevels`（ops/maxAnswer）をステージデータに持たせ毎ラウンドランダム生成。Lv1=和≤5の足し算/Lv2=答え9まで/Lv3=足し算＋引き算（答え≥1のみ）。記録は既存 mathStats（'5-2'等のキー）
- **⑧ ライフ制**: 全ステージ共通 lives=3・ハート3つを右上HUDに常時表示。減るのは**誤答ショットのみ**（撃ち逃し・時間切れは減らない=統計ルールと同思想）。ハートは割れずに🌫️（もやもや）に包まれる。残り1（=2ミス）で正解グロー必発（負ける前に助け舟）。0で `stage-failed` → React の失敗画面「もやもやに つつまれちゃった！」（暗転なし・もういちど大ボタン・学習記録は誤答時点で保存済み）。難易度開始ごと（シーン再マウント）に全快
- 課題メモ: 2Dの配置グリッドが3×2=6スロットのため targetCount 上限6。🌫️絵文字は環境によりグレー雲の見た目になる（実機Android/iOSでは絵文字表示）

## もどるボタン＋難易度3段階＋カタカナこうえん（2026-07-08）

- **もどるボタン**: ゲーム/リザルト画面の左上に常時表示（DOM層・safe-area対応・58px）。ゲーム中は `game-pause` でシーンを一時停止して「⏸やめる？ ▶つづける/🗺️やめる」を挟む
- **難易度3段階**: 各ステージ Lv1→2→3 の順に解放。2.5D面: Lv2=似た文字を必ず混ぜる（letterStats の苦手スコアでペアを選ぶ・固定羅列にしない）＋プール開放+3 / Lv3=さらに選択肢+1（最大6・アーチ配置は選択肢数から動的生成）＋テンポ約15%アップ。2D面: Lv2/3=ターゲット+1/+2・速度+12%/24%。70〜85%帯セーフティ・撃ち逃し非記録・オートエイムは全難易度共通
- **進捗 schemaVersion 3**: `stageLevels`（ステージ×難易度のクリア状況）を追加。v2→v3移行=★のあるステージはLv1クリア扱い。解放判定はステージデータの `unlock: {stageId, minLevel}` から導出（旧 unlockedStages 所属はロックアウト防止でそのまま解放）
- **マップ**: カードに難易度バッジ 1/2/3（クリア=★金・次挑戦=紫ふちどり）。タップで次挑戦難易度を自動選択。リザルトの「つぎ」=同ステージ次レベル→Lv3後は次ステージ
- **名称変更**: もじシティこうえん「あ」→ ひらがなこうえん（ID `hiragana-a` は不変）
- **カタカナこうえん**（`katakana-a`）: ひらがなLv3クリアで解放。同エンジン・カタカナプール。`KATAKANA_CONFUSABLES`（シ/ツ・ソ/ン・ク/ワ等）と音類似グループのカタカナ版を追加。統計は既存 letterStats 構造（ラベルがカタカナなので衝突しない）
- 課題メモ: 2D面の Lv2/3 はターゲット数と速度のみ（word/number/math の専用難化はフェーズ2で検討）。ね/れ等の音類似ペアは引き続き選択肢に同時に出ない（音声のみ出題のため）

## 連戦→ボス化（2026-07-04）— レビューでの設計判断

1. **ザコは正解1回で浄化**（メーター非表示・即テンポ）、**ボスだけ複数段メーター** — ザコにも3回要求すると1ステージが長すぎ、4歳の集中が持たない。1体あたりの正解数は `battle.purifyStepsPerEnemy` で変更可能。
2. **狙う文字は3重提示**（音声＋中サイズの文字フラッシュ＋ミッションバー常時表示） — 敵ごとに文字が変わるため、TTSが鳴らない端末でも迷子にならないように。
3. **プールは poolStart から徐々に開放**（correct 2回以上で1文字ずつ） — 最初から全ひらがなは過負荷。**ボスは直近で練習した文字のみ**で「学んだ成果で締める」。
4. **正答率70〜85%帯の自動調整を実装** — セッション正答率>85%で類似文字（CONFUSABLES）混入、<70%で選択肢を1つ削減。
5. 選択肢はひらがなステージではひらがなで統一（数字・カタカナ混在をやめた）。

## 2.5D 化フェーズ1 完了（2026-07-04）— レビューでの設計判断

1. **移動区間は5〜6秒に短縮＋移動中も自由撃ちOK（無罰）** — 「進む」が待ち時間になると4歳は離脱するため。
2. **対峙中はカメラほぼ完全停止** — 背景が動き続けると文字が読みにくく酔いやすい。バブは±3px→対峙中0.8pxに減衰。ヒットシェイクも2D版より弱めた。
3. **選択肢バブルはスクリーン空間固定サイズ・不透明・最前面** — 奥行き演出（拡縮）を文字に適用しない。照準リングも文字より下のレイヤー。読みやすさ＝学習の生命線。
4. **撃ち逃し・時間切れは記録しない** — 時間切れそのものを作らず、選択肢は答えるまで待つ（12秒で音声再提示・22秒で正解が光るやさしいヒントは維持）。
5. **次に狙う文字は `src/learning/picker.ts`（簡易間隔反復）が選ぶ** — wrong多・未見・低mastery優先。ステージ1はプールが「あ」のみなので実質固定だが、フェーズ2でプールが増えると効く。
6. **右下の発射ボタンは見送り（設計上書き）** — レイアウトAの学習タスクは「どのバブルを選ぶか」であり、タップ＝選択＋発射が最短。ボタンだけでは文字を選べず学習が成立しない。フェーズ3の手動照準モード導入時に再検討。
7. **依頼文にある「既存の親レポート・設定画面・難易度調整・間隔反復」はフェーズ3項目で未実装** — 壊してはいないが「再利用」ではなく今後の実装。カメラ快適設定はやさしい既定値（揺れ:すこし/自動照準:ON/速度:ゆっくり）を定数で実装済み、設定UIはフェーズ3。

## フェーズ1で仕様に対して行った設計判断（レビュー反映済み）

1. **ビームはヒットスキャン方式** — 弾が飛ぶ方式だと着弾まで遅延が出て「押した瞬間の快感」と矛盾するため、タップした瞬間に判定・着弾演出を発火し、ビームは一瞬の視覚演出とした。
2. **出題文字はミッションバーに常時表示＋音声併用** — TTS フォールバックが常時機能し、音と形の同時提示が定着にも効くため。🔊ボタンでいつでも再読み上げ可能。
3. **学習統計（LetterStats）はフェーズ1から記録** — フェーズ3の間隔反復・難易度調整が後付けにならないよう、seen / correct / wrong / 平均反応時間 / masteryLevel を最初から蓄積。
4. **空撃ちは無罰** — 何もない所を撃ってもキラッと光るだけ。連打の楽しさを削がない。
5. **やさしい救済はフェーズ1から** — 誤答2回で正解を大きく・全体をゆっくり、誤答3連続で正解を一度だけ光らせる、12秒無操作で音声再提示、22秒でヒント発光。

## 本番化に向けた改善候補（優先度順）

1. **録音音声への差し替え** — Web Speech API は端末差が大きい。短い録音ファイル（あ〜ん・掛け声）に差し替えるのが品質向上の最短ルート。`src/audio/voice.ts` の置き換えだけで済む構造にしてある。
2. **BGM** — 現状効果音のみ。ループする軽い BGM（WebAudio 合成 or 短いループ音源）と設定画面の ON/OFF。
3. **ヒーロー選択** — 画像は5色分あり `HEROES` データも定義済み。タイトル→ヒーロー選択画面を足すだけ。
4. **バブル同士の重なり回避の改良** — 現在は簡易な押し戻しのみ。長時間で軌道が単調になることがある。
5. **休憩促し** — 連続プレイ15〜20分で「きゅうけいしよう」演出（仕様 §10）。
6. **Phaser のバンドルサイズ** — 1.6MB(gzip 425KB)。dynamic import でタイトル表示を先行させると初回体験が良くなる。
7. **効果音の質** — WebAudio 合成は軽量だが素朴。本番では録音 SE への差し替えを推奨。
8. **正解演出のバリエーション** — 毎回同じだと飽きる。コンボ数に応じた演出強化（花火・虹ビーム等）。
9. **誤答の混同ペア記録** — 現在は出題文字への wrong のみ。どの文字と間違えたか（あ↔お 等）を記録すると親レポートの「見分け練習中」がより正確になる。
10. **アクセシビリティ** — 効果音だけに依存する情報がないかの点検、色覚特性への配慮（バブル色は情報を持たせていないので現状問題なし）。
